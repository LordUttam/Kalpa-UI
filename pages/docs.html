Button

A button is one of the most important part of a website or app. It is the thing on which user will click to do perform action. Hence also called CTA. The base styles for button are in class btn.

Primary button

Primary button is the main button which performs primary actions of your app. You can use btn for button and for particular button use btn--primary to use it, as shown below.

Primary outline button

Primary outline button is the second most used button in many websites. This button only has an outline and the background is transparent. You can use btn for button and for outline button use btn--outline, as shown below.

Link button

Link button is used in places you want a button it is not a primary action and it links to a different page or section.

Button with icon

Many times we use an icon inside button. This can simply be done by placing icon in the button tags.

Floating action button

A floating action button (FAB) performs the primary, or most common, action on a screen. It appears in front of all screen content, typically as a circular shape with an icon in its center. You can use btn--float as shown below.

Image

Images are the most important component of a website as they attract the user's attention and, therefore, should be responsive.

Responsive Image

Add the class img--responsive to create a responsive image to your website. The width and height scale down depending on the size of the parent. Resize the container below to see how it works.

Round Image

Add the class img--round to make an image round. It will maintain its aspect ratio.

Input

Inputs allow users to enter information and they can be of different types - text, number, password, email, submit, etc.

Text Input

Textbox inputs are generally used in forms for the entry of the data. This is the normal textbox where user will enter the details like name etc.

Text Input with inline label

Adding a class in container element input input__container--row will make the input and label appear in single line.

Successful Input

If the user enters correct data, successful input can be shown by adding input--success class. This will also make the input__message element directly next to it appear in success color.

Input Successful

Error Input

If the user enters incorrect data (mistakes happen), error input can be shown by adding input--error class. This will also make the input__message element directly next to it appear in error color.

Bad Input
